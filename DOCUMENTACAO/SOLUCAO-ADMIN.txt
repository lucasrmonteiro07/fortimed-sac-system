════════════════════════════════════════════════════════════════════════════

O campo password_hash é OBRIGATÓRIO na tabela public.users

Solução: Usar crypt() para gerar um hash seguro da senha


🚀 EXECUTE AGORA
════════════════════════════════════════════════════════════════════════════

No Supabase SQL Editor, cole e execute:

INSERT INTO public.users (
    id,
    email,
    password_hash,
    name,
    role
) VALUES (
    '2e6d9014-8fdf-4e5a-b41f-ac7e2c2ea53f',
    'anapaula@fortimeddistribuidora.com.br',
    crypt('Compras@01', gen_salt('bf')),
    'Ana Paula',
    'admin'
);
